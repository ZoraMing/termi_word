# Termi Word 3

一个专为终端用户打造的摸鱼/高效背单词工具，基于 Textual TUI 框架，内置 FSRS 科学记忆算法。

## 特性

- **命令行原生**：不用起重型客户端，分出终端小窗就能默默背词。
- **科学抗遗忘**：基于 FSRS 算法，自动调整复习间隔。
- **窗口动态自适应**：界面能在 `6-16` 行高间弹性适配。长词义和例句自动折行，支持 `↑↓` 方向键平滑滚动。
- **顺畅无延迟**：重绘逻辑全部走内存缓存，消除界面渲染时的 SQLite 查询卡顿。
- **全键盘操作**：双手无需离开主键盘区，所有功能一键直达。
- **完全本地离线**：本地 SQLite 存储，不需要网络连接。
- **自定义任何词包**：直接将 CSV 词书放入 `data/imports/` 即可导入，支持自定义列名映射。
- **拼写练习**：支持听音拼写，打字时顶部提供实时进度与对错反馈。
- **打卡日历**：极简月历视图，按月拉取历史数据，轻量高性能。

## 快捷键与使用指南

### 1. 全局快捷键

| 快捷键 | 功能 | 说明 |
|:---|:---|:---|
| `Ctrl + /` (或自定义) | 打开全局单词搜索 | 可以在任意界面唤起搜索，返回后自动关闭 |
| `Ctrl + Z` / `Ctrl + Q` / `Ctrl + C` | 安全退出 | 双击 `Esc` 也可以安全退出，退出前会自动保存数据并释放连接 |
| `Esc` | 返回上一级 / 取消锁定 | 用于从子页面退回主页面或取消编辑/锁定状态 |
| `?` (或 `Shift + ?`) | 显示/隐藏底部快捷键栏 | 可以在大多数页面上切换底部帮助信息的可见性 |

### 2. 主页面 (首页)

展示今日学习目标（新词、复习、拼写进度）与已完成统计。

| 快捷键 | 功能 |
|:---|:---|
| `1` | 开始学习（今日新词学习 + 待复习单词） |
| `2` | 仅复习（不背新词，仅复习已到期卡片） |
| `3` | 拼写练习 |
| `4` | 词表浏览与检索 |
| `5` | 学习日历与计划配置 |
| `6` | 快捷键与选项设置 |
| `8` | 词本管理与导入 (CSV 映射配置) |

---

### 3. 学习/复习页面

采用科学的间隔重复记忆系统（FSRS）。

| 快捷键 | 功能 | 说明 |
|:---|:---|:---|
| `Space` / `Enter` | 翻卡 / 确认评分 | 单词正面按此键可翻转展示背面释义与例句 |
| `1` | 陌生 (Again) | 没记住，重新学 |
| `2` | 熟悉 (Hard) | 勉强记住，下次复习间隔较短 |
| `3` | 记得 (Good) | 顺利记住（推荐），复习间隔正常 |
| `4` | 掌握 (Easy) | 牢固掌握，复习间隔较长 |
| `s` | 跳过 (Skip) | 临时跳过当前词，不记录评分与进度 |
| `t` | **挂起单词 (Suspend)** | 永久停用该词。**需再次按 `y` 确认，`Esc` 取消**。挂起后从当前会话队列剔除，并且未来的任何背诵和拼写练习都不再调度该词。 |
| `f` | **收藏/取消收藏 (Star)** | 切换收藏状态（卡片右上角将标记 `*`），便于后续单独过滤复习。 |
| `↑` / `↓` | 垂直滚动 | 释义或例句过长溢出面板时，按上下方向键平滑滚动查看全部内容 |
| `Esc` | 返回 | 中途返回主页面，系统会自动保存当前已完成的背诵进度 |

### 4. 拼写评测页面

用于检验拼写记忆。支持听音、看音标与词性进行手动拼写拼词。

| 快捷键 | 功能 | 说明 |
|:---|:---|:---|
| `字母键` | 拼写输入 | 在输入框中输入拼写内容，顶部有实时进度条反馈 |
| `Enter` | 提交拼写 / 下一词 | 输入完单词后按回车提交校验。**已判定对错后，再次按回车切换到下一题** |
| `Tab` | **获取提示 (Hint)** | 每按一次，在输入框中自动提示单词的下一个正确字母，用于辅助拼写 |
| `Space` | **显示答案 (Show)** | **仅在输入框为空时有效**，直接显示正确答案，但不会提交计分 |
| `s` | **跳过单词 (Skip)** | **仅在输入框为空时有效**，直接跳过并切换到下一题 |
| `Esc` | 返回 | 放弃本次拼写并返回主页面 |

### 5. 词表浏览与搜索页面

展示当前词本的所有单词，支持关键字实时模糊搜索。

| 快捷键 | 功能 | 说明 |
|:---|:---|:---|
| `↑` / `↓` | 移动光标 | 在单词列表中上下移动选择单词 |
| `f` / `Ctrl + F` | 切换仅看收藏 | 切换是否仅过滤已收藏的单词（输入框聚焦时必须使用 `Ctrl+F` 触发） |
| `Enter` / `Space` | 锁定底部详情 | 将当前单词的详细释义卡片锁定。**锁定后可按 `↑` `↓` 键滚动阅读超长内容**。再次按该键或 `Esc` 解锁 |
| `Backspace` | 删除搜索字符 | 更改搜索输入内容 |
| `Esc` | 退出 | 退出搜索框焦点 / 退出详情锁定 / 返回主页面 |

### 6. 学习日历与计划配置页面

以月历形式查看历史背词记录与打卡标记，并支持修改每日计划目标。

| 快捷键 | 功能 | 说明 |
|:---|:---|:---|

| `数字 + Enter` | 提交修改 | 确认并保存新计划目标数字 |
| `Esc` | 取消修改 / 返回 | 编辑状态下按 `Esc` 放弃修改并退出输入；非编辑状态下按 `Esc` 返回主页面 |

### 7. 词本管理与字段映射页面 (主页面按 `8` 进入)

支持在不同的本地 CSV 词书之间切换，并允许用户自由绑定 CSV 列名与内部单词字段的映射。

| 快捷键 | 功能 | 说明 |
|:---|:---|:---|
| `↑` / `↓` | 选择项目 | 在可用词书列表和列映射映射项之间移动 |
| `Enter` / `Space` | 切换/修改绑定 | 1. 选中 CSV 词书时按此键：**再次按 `y` 确认切换并全量导入**该词书。<br>2. 选中列映射项（如：单词映射、中文映射）时按此键：**可在该 CSV 包含的所有列名中循环切换映射**，映射完毕后移动到最下方选择 `确认执行同步` 并按回车。 |
| `y` | 确认操作 | 确认执行词包切换、同步导入或映射更改 |
| `Esc` | 返回 / 取消 | 取消当前的二次确认状态，或退回主页面 |

## 如何导入自己的词包 CSV

你可以自己做词本。把它丢到运行目录下的 `data/imports/` 里，程序启动时会自动扫到并支持一键切书。

### 1. 推荐的 CSV 表头 (开箱即用)

如果你的 CSV 文件表头（第一行）使用以下列名，TermiWord 会自动识别并完美匹配，无需手动映射配置：

| 列名 | 说明 | 示例值 | 是否必须 |
|:---|:---|:---|:---|
| `w` | 单词 (Word) | abandon | **必须** |
| `zh` | 中文释义 (Chinese) | vt. 放弃，遗弃 | 推荐 (用于背诵时查看) |
| `en` | 英文释义 (English) | to leave behind | 可选 |
| `us` | 音标 (Pronunciation) | /əˈbændən/ | 可选 |
| `c` | 词性分类 (Category) | 核心词汇 | 可选 |
| `core` | 核心词标记 (Core Tag) | 1 (1表示核心，0或空表示普通) | 可选 |
| `ex` | 英文例句 (Example) | He abandoned his car in the snow. | 可选 |
| `exz` | 例句翻译 (Example Trans) | 他把车丢弃在雪地里。 | 可选 |

### 2. 自定义 CSV 列名映射 (兼容任何 CSV 格式)

如果你的 CSV 词书表头列名与推荐的不一致（例如使用的是 `Word` 代替了 `w`，`Translate` 代替了 `zh`），按以下步骤导入：

1. **准备 CSV 文件**：
   - 确保 CSV 文件为 **UTF-8 编码**（如果是 Excel 另存的 CSV，请确保用记事本或其他工具转换为 UTF-8 编码，防止乱码）。
   - 确保第一行为表头。
   - 将文件拷贝至 `data/imports/` 目录下（如 `my_words.csv`）。
2. **在应用中绑定字段**：
   - 启动 TermiWord，在主页面按 `8` 进入 **词本导入与配置** 界面。
   - 在顶部的 CSV 列表中找到你的文件 `my_words.csv`，按回车将其设为当前词书。
   - 在下方的 “映射关系配置” 列表中，按 `Enter` 或 `Space` 循环切换，把 `单词映射(w)` 绑定到你 CSV 中的对应列（如 `Word` 列），把 `中文映射(zh)` 绑定到 `Translate` 列，以此类推。
   - 映射配置完成后，移动光标到最下方的 **`[确认执行同步]`**，按回车。
   - 再次按 **`y`** 确认，系统将在后台异步读取该 CSV 词书并执行增量数据清洗和入库。

## 配置文件说明

配置文件位于运行目录下的 `data/ui_config.json`，在修改“学习日历”里的目标数量或快捷键时，会自动回写此文件：

```json
{
  "active_deck": "IELTSluan_2",
  "daily_new_target": 20,
  "review_soft_limit": 50,
  "daily_spelling_target": 15,
  "spelling_enabled": true,
  "search_shortcut": "ctrl+slash",
  "panel_max_width": 120,
  "panel_min_height": 6,
  "panel_max_height": 16
}
```

* `active_deck` - 当前处于激活状态的词书名
* `daily_new_target` - 每日学习新词计划数量
* `review_soft_limit` - 每日复习软上限数量
* `daily_spelling_target` - 每日拼写测试计划数量
* `spelling_enabled` - 是否开启拼写模块
* `search_shortcut` - 全局单词搜索唤起按键（默认 `ctrl+slash`）
* `panel_max_width` - 核心显示面板最大限制宽度（默认 120 字符）
* `panel_min_height` / `panel_max_height` - 物理自适应终端的最小与最大行高限制

### 自定义文件位置

如果找不到词书或配置文件，可以在主页面按 `6` 进入设置页，修改：

| 设置项 | 默认位置 | 说明 |
|:---|:---|:---|
| `数据目录` | `data/` | 存放数据库、配置文件，`imports/` 子目录自动创建 |

路径可以使用绝对路径，也可以使用相对路径。相对路径会按应用运行目录解析，例如：

```text
E:/termi-word-data
./my_data
```

修改后会写入固定引导文件 `data/paths.json`：

```json
{
  "config_dir": "E:/termi-word-data"
}
```

程序会自动在您指定的数据目录下建好 `imports/` 文件夹供您放词书。

`paths.json` 必须留在可执行程序旁的 `data/` 目录里，不然程序启动时找不到引导数据。

> **热更新**: 路径在设置页修改完保存后会**立即生效**，不需要重启程序。后台会自动释放老 SQLite 引擎连接，无缝在新目录下重载配置、重建数据库并绑定各服务。

## Nuitka 打包指南

我们使用 Nuitka 的 `onefile` 模式把程序打包成单个独立的可执行文件。注意，因为底层依赖差异，不支持交叉编译（也就是说你得在 Windows 上构建 `.exe`，在 macOS 上打包 Unix 可执行文件，在 Linux 下构建 ELF 文件）。

### 打包产物结构

```text
./termi_word_package/           # 默认位于当前执行路径的 pwd 下
├── termi-word(.exe)            # 单文件可执行程序
└── data/
    └── imports/                # 放置待导入 CSV 词书
        └── *.csv
```

首次运行时，程序会自动在 `data/` 目录下创建：
- `termi_word.sqlite3` - 本地数据库
- `ui_config.json` - UI 配置文件
- `local_time.json` - 时区配置

### 快速打包步骤

**1. 安装依赖**

```bash
pip install -r requirements.txt
```

**2. 执行打包**

Windows (PowerShell):

```powershell
.\scripts\build.ps1
```

macOS / Linux:

```bash
sh ./scripts/build.sh
```

**3. 自定义打包目录**

默认打包工作目录为当前执行路径下的 `termi_word_package/` 目录，可通过参数自定义修改打包输出路径：

```powershell
# Windows (直接按位置传入自定义打包目标路径)
.\scripts\build.ps1 "D:/my_package"

# macOS/Linux
PACKAGE_ROOT="/path/to/my_package" sh ./scripts/build.sh
```

### 打包工作目录说明

```text
E:/termi_word_package/
├── termi-word.exe     # 最终可执行文件（从 output/ 复制）
├── output/            # Nuitka 编译产物
├── cache/             # Nuitka 下载缓存和编译缓存
├── temp/              # 编译临时文件
├── local-appdata/     # 隔离的 LOCALAPPDATA（防止污染系统目录）
└── data/imports/      # 自动从项目 data/*.csv 复制
```

### Conda 环境支持

构建脚本会自动检测 Conda 环境，并打包 `ffi*.dll` 和 `sqlite3.dll` 以防止 ctypes/sqlite3 导入崩溃。

### 使用打包后的程序

```bash
# 直接运行
./termi-word          # macOS/Linux
.\termi-word.exe      # Windows

# 将 CSV 词书放入 data/imports/ 目录
cp my_words.csv data/imports/

# 启动后按 8 进入词本管理，导入词书
```

### 提示

- 打包后的最终产物是**单文件**，拷到其他电脑上就能跑，完全不用解压。
- 首次跑起来的时候，程序会在它旁边自动把 SQLite 数据库和配置文件生成好。
- 如果想把数据存放在别的地方，跑起来后去设置（按 `6`）改“数据目录”和“词书目录”即可。
- 打包时产生的大量中间文件夹可以在完工后安全删掉，只要留着 `termi-word.exe` 以及它旁边的 `data/` 目录就行。
